UC Browser

UC Browser is a proxy browser that focuses on both decreasing bandwidth. Pages are requested by their servers, but rendered on the device.

Summary

Data Savings Up to 60%
HTTPS Behavior Bypasses proxy
Network Optimization Not sure
Transcoding/Compressing Images Yes
Content compression Yes, if Cloud Boost is on.
X-Forwarded-For Header Yes
Additional Headers HTTP_X_UCBROWSER_UA
Cache-control: no-transform Not sure
Rendering Behavior Done on client
Rendering Engine WebView

Basic Architecture

When the client makes a request, the request is routed through UC's servers if Cloud Boost is enabled. The servers perform the initial DNS resolution. They also initiate each request over their own networks, optimize the response, and then pass back to the client. The pages are rendered on the client, and all JS execution takes place on the client.

Gotchas

  • Using WebView
  • Does not appear to support @media with no declared type
