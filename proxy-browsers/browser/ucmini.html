UC Mini

UC Mini is a proxy browser that focuses on both decreasing bandwidth and reducing impact on devices. Pages are requested by their servers, and rendered on the server.

Summary

Data Savings Up to 90%
HTTPS Behavior Still routes through proxy
Network Optimization Not sure
Transcoding/Compressing Images Yes
Content compression Yes, if Speed Mode is on (default behavior).
X-Forwarded-For Header Yes
Additional Headers HTTP_X_UCBROWSER_UA, HTTP_X_UCBROWSER_DEVICE_UA
Cache-control: no-transform No
Rendering Behavior Done on server
Rendering Engine U3 or U2 Kernel, based on WebKit

Basic Architecture

When the client makes a request, if Speed Mode is disabled, the browser simply interacts with a website in the traditional manner, making use of a WebView. If Speed Mode is enabled (default mode) the request is routed through UC's servers. The servers perform the initial DNS resolution. They also initiate each request over their own networks (sending an X-Forwarded-For header). The pages are rendered on the server, and all JS execution takes place on the server.

The page is sent back to the client in a proprietary format. Whenever something happens that may change the DOM or Render Tree, the client makes a request to the server. The server sends back the updated snapshot of the page for the client to display.
