Chrome Data Compression

Chrome's Data Compression Proxy is a conservative proxy (comparitively) that focuses on the network and leaves rendering entirely to the client. The optimizations are performed by the PageSpeed Module tuned to mobile devices. Specifically, it uses the Optimizing for Bandwidth" module.

Basic Architecture

When the client makes a request, the request is routed through Google's optimization servers (unless it's HTTPS, in which case the site is loaded directly through the device as normal). The servers perform the initial DNS resolution. They also initiate each request over their own networks (sending an X-Forwarded-For header), optimize the response, and then pass back to the client.

If possible, the proxy will run over SPDY (basis for HTTP/2.0) which allows the proxy to allows for multiplexing of request and parallel response streams.
